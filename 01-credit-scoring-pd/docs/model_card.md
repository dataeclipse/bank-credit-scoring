# Model Card — PD Scoring (кредитный скоринг)

> Финальная карточка модели в духе банковской валидации (НБРК / Fed **SR 11-7** / Basel).
> Версионируется вместе с моделью.

| Поле | Значение |
|---|---|
| Модель | PD scoring (вероятность дефолта по кредитной заявке) |
| Прод-модель | **LightGBM**, MLflow registry `pd-lightgbm` **v3** (alias `@champion`) |
| Версия кода | pd-scoring **1.0.0** |
| Дата | 2026-06-30 |
| Владелец / валидатор | dataeclipse (DS) / требуется независимая валидация перед прод |
| Статус | Релиз v1.0.0 (демо/портфолио; не боевое кредитное решение) |

## 1. Назначение и область применения (Purpose & Scope)
Оценка PD по кредитной заявке для **поддержки** решения о выдаче (decision support), не автоматический отказ.
- **In-scope:** потребительское кредитование физлиц, скоринг на этапе заявки по данным анкеты + (в полном
  контуре) агрегатам кредитной истории; ранжирование и сегментация риска, reason codes для обоснования.
- **Out-of-scope:** автоматический отказ без участия человека; юрлица/ипотека/иные продукты; использование
  как единственного основания решения; перенос на иную популяцию без ревалидации.

## 2. Данные
- Источник: **Home Credit Default Risk** (Kaggle) — анонимизированные данные потребкредитования
  (близко к рынку РК). Целевая переменная `TARGET` (1 = дефолт).
- Витрина: **356 255** клиентов × **120 фич** (curated application + engineered ratios + client-level
  агрегаты bureau/bureau_balance/previous/installments/POS/credit_card), агрегации **без утечек будущего**.
- Баланс классов: **8.07%** дефолтов (сильный дисбаланс ~1:11) — учтено в метриках (PR-AUC/KS/Gini).
- Известные особенности: пропуски (EXT_SOURCE, агрегаты карт ~71%), аномалия `DAYS_EMPLOYED=365243`
  (18%, вынесена во флаг). Подробно: [eda.md](eda.md), словарь — [data_dictionary.md](data_dictionary.md).
- **Ограничения данных**: анонимные прокси (внешняя валидность ограничена); только одобренные заявки
  (survivorship — см. §8); датасет не РК-специфичен.

## 3. Методология
- **Split**: stratified hold-out по TARGET, seed 42 (train 246 008 / holdout 61 503). HPO (Optuna)
  на подвыборке train, финальные модели — на полном train.
- **Scorecard**: WOE/IV-биннинг (`optbinning`, биннинг внутри CV-фолдов — без оптимизма) + логистическая
  регрессия → баллы; отбор фич по IV≥0.02 (76 из 120).
- **GBDT**: LightGBM и CatBoost (нативные категориальные/пропуски), early stopping, фиксированные сиды.
- **Выбор в прод — LightGBM** (Gini 0.579 ≈ CatBoost, выше scorecard на Δ Gini 0.04); scorecard оставлен
  как интерпретируемый challenger. Балл — PDO/odds-шкала (base 600, PDO 50). Обоснование:
  [model_selection.md](model_selection.md).

## 4. Метрики качества
Holdout 61 503 заявок (Фаза 1 stratified split, seed 42). HPO — Optuna на подвыборке train,
финал — на полном train. Выбор в прод: **LightGBM** (scorecard — интерпретируемый challenger).

| Метрика | Scorecard | LightGBM | CatBoost |
|---|---|---|---|
| ROC-AUC | 0.770 | **0.790** | 0.789 |
| Gini | 0.539 | **0.579** | 0.579 |
| KS | 0.407 | **0.440** | 0.440 |
| PR-AUC | 0.255 | 0.287 | 0.289 |

Scorecard CV (биннинг внутри фолдов, без оптимизма): Gini 0.528. Детали:
[model_comparison.md](model_comparison.md), [model_selection.md](model_selection.md).

## 5. Калибровка
Калибратор обучен на отдельном calib-наборе (из train), оценка на holdout — без утечки.
**LightGBM уже хорошо откалиброван**: raw Brier **0.0657**, ECE **0.0028** (< 0.01). Isotonic/Platt
не улучшают (isotonic ECE 0.0042, sigmoid 0.0148) → в прод можно отдавать raw PD; isotonic-калибратор
сохранён как страховка. Детали и reliability curve: [calibration.md](calibration.md).

## 6. Объяснимость
SHAP **TreeExplainer** по прод-LightGBM. Глобально важнейшие фичи: **EXT_SOURCE_2/3/1**, просрочки и
долг бюро, история платежей. Локальные **reason codes** — топ-факторы «за/против» по заявке (знак SHAP =
направление влияния на риск), маппинг фич → человекочитаемо. Примеры + beeswarm: [explainability.md](explainability.md).

## 7. Справедливость (Fairness)
Прокси-группы (анонимные): **CODE_GENDER**, возрастные бэнды. Порог решения — KS-точка.
- **Пол**: disparate impact (approval) **0.816** (≈ граница 4/5), equalized odds diff 0.126 — приемлемо.
- **Возраст**: disparate impact **0.627** (< 0.8) — молодые (<30) одобряются заметно реже; equalized odds diff 0.371.
Митигация не применялась (диагностика); опции — reweighting, ThresholdOptimizer, групповая калибровка.
Детали: [fairness.md](fairness.md).

## 8. Ограничения и риски
- **Чёрный ящик** (GBDT) — снимается SHAP reason codes; scorecard как интерпретируемый challenger.
- **Bias по возрасту** (DI 0.627) — мониторить и обосновывать; молодые заёмщики в зоне риска отказа.
- Прокси-группы **анонимны** (Home Credit) — внешняя валидность ограничена.
- **Reject inference** не делался — модель училась на одобренных (survivorship bias). Опция на будущее.
- Калибровка зависит от стабильности распределения входов — см. мониторинг дрейфа (Фаза 4).

## 9. Требования к мониторингу
- **PSI/CSI** по входным фичам и **дрейф PD** (Evidently + собственный PSI). Порог алерта **PSI > 0.2**
  (эмпирически: <0.1 нет сдвига, 0.1–0.2 умеренный, >0.2 значимый). Инструмент: `pd-scoring-drift`.
- **Периодичность**: дрейф — на каждой батч-партии/ежедневно; ревалидация модели — не реже раза в
  6–12 мес. или при срабатывании алерта дрейфа / падении Gini на свежем срезе.
- Метрики стабильности скорборда (PSI скорбалла), доля review/approve, fraud/default rate во времени.
- Деградация качества (Gini/KS на размеченных свежих данных) → триггер ревалидации/переобучения.
- Детали и демонстрация «ловли» дрейфа: [monitoring.md](monitoring.md).

- **Реестр**: MLflow Model Registry; прод-версия по алиасу `pd-lightgbm@champion` (v3, Gini 0.5789).
  Схема витрины версионирована (`feature_schema.json`, hash `ac4281b5`).
- **Воспроизводимость**: фиксированные сиды, `uv.lock`, артефакты обучения в MLflow; контейнер запекает

- **Версия модели меняется** при переобучении/смене фич → новая версия в registry + обновление этой карточки.
