# Load test — латентность /score (Фаза 4)

Сервис: 1 uvicorn-воркер, CPU, прод-LightGBM (120 фич) + native TreeSHAP reason codes.

## Одиночный запрос (без contention, 100 послед. запросов)
| Метрика | Значение |
|---|---|
| p50 | **83 мс** |
| p95 | **86 мс** |
| p99 | **86 мс** |
| mean | 85 мс |

Это честная латентность обработки заявки (build row → predict_proba → pred_contrib → reason codes).

## Бюджет латентности (разложение p99 ≈ 86 мс)
In-process замер компонентов `score()` (300 итераций, без HTTP):

| Этап | мс | доля |
|---|---|---|
| `pred_contrib` (reason codes, TreeSHAP) | 38.8 | **45%** |
| `build_feature_row` (сборка 120-фич DataFrame) | 28.7 | **33%** |
| `predict_proba` (inference PD) | 11.9 | 14% |
| сериализация ответа (pydantic) | ~0.01 | ~0% |
| **score() итого** | **86.6** | 100% |

HTTP-обёртка (FastAPI/uvicorn) добавляет пренебрежимо мало: p50 по HTTP (83 мс) ≈ in-process score().
**Главные статьи — reason codes (45%) и сборка строки (33%)**, не сам inference.

## Опции ускорения (зафиксировано, не внедрено)
- **Reason codes по требованию** — флаг запроса/отдельный эндпоинт `/explain`: без объяснения
  `score()` ≈ 48 мс (−39 мс), объяснение считать только когда нужно.
- **Оптимизация build_row** — собирать вход сразу в `numpy`-массив в нужном порядке вместо
  `DataFrame` + поколоночного `to_numeric` (потенциально −20 мс).
- **Предрасчёт для типовых сегментов** / кэш по хэшу нормализованной заявки.
- **Батч-скоринг** для офлайн-партий (амортизация overhead на запрос).
- **Горизонтальный масштаб** под нагрузкой: `uvicorn --workers N` / реплики (см. ниже).

## Под нагрузкой (locust, 20 пользователей, 15с, 1 воркер)
- Пропускная способность ≈ **17 req/s**, 0% ошибок.
- p50 ≈ 1100 мс — **насыщение одного воркера** (sync CPU-bound эндпоинт сериализуется под GIL),
  а не рост стоимости запроса.

## Запуск
```bash
make run                     # поднять сервис на :8000
uv sync --extra loadtest
uv run locust -f loadtest/locustfile.py --headless -u 20 -r 10 -t 15s --host http://localhost:8000
```
(Windows: запускать с `PYTHONUTF8=1` — locust читает pyproject.toml.)

## Вывод
Per-request латентность ~85 мс приемлема. Пропускная способность масштабируется горизонтально:
`uvicorn --workers N` или реплики за балансировщиком (Фаза 5, Docker/compose).
